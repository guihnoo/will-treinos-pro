# Skill: Auditing Security — Will Treinos PRO

## When to use
- Before any production deploy
- When adding a new Supabase table
- When creating a new API route
- When adding file upload
- When changing authentication logic

## OWASP Top 10 adapted for Will Treinos PRO

### A01 — Broken Access Control
```
Check:
□ middleware.ts protects /admin and /coach routes?
□ Client components with user.role only check on UI (correct)?
□ Supabase queries have RLS active on the table?
□ Student cannot see other students' data?
□ Coach cannot access admin financial data?
```

### A02 — Cryptographic Failures
```
Check:
□ SUPABASE_SERVICE_ROLE_KEY without NEXT_PUBLIC_ prefix?
□ NEXT_PUBLIC_SUPABASE_ANON_KEY is the only public key?
□ Sensitive cookies with HttpOnly + SameSite=Strict + Secure?
□ wt_role cookie cannot be manipulated by client?
```

### A03 — Injection
```
Check:
□ Supabase queries use parameters (not string concatenation)?
□ Text inputs sanitized before saving?
□ Filenames generated by UUID (never from user)?
```

### A04 — Insecure Design
```
Check:
□ OAuth gate of 45min active (WT_STAFF_OAUTH_OK)?
□ Check-in has time window (60min before class)?
□ External XP anti-cheat (10 XP vs 50 XP)?
```

### A07 — Auth & Session Failures
```
Check:
□ Supabase session expires correctly?
□ Logout clears all cookies and localStorage?
□ Staff OAuth requires row in staff_access with email?
```

### A08 — File Upload
```
Check:
□ MIME type validated (not just extension)?
□ Maximum size defined (recommended: 5MB for profile photo)?
□ Filename replaced by server-generated UUID?
□ Upload goes to private Supabase Storage bucket?
```
